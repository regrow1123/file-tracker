# 2026-03-11 파일변경추적 프로젝트 세션 기록

## 프로젝트 개요
- 대규모 리눅스 서버(수백대) 환경에서 /home 이하 파일 삭제/mtime 변경 실시간 추적
- eBPF 기반, 파일시스템 벤더 무관 (VFS 레벨)
- RHEL9 + RHEL10 동시 지원 (CO-RE 단일 바이너리)
- uid 추적 불필요, 파일 전체경로만
- Kafka로 이벤트 전달, WAL로 at-least-once 보장

## 기술 결정
- 언어: C++ (유저스페이스) + C (BPF probe)
- BPF: kprobe (vfs_unlink, vfs_write, do_truncate, vfs_utimes)
- 경로 해석: dentry 컴포넌트 배열 방식 (verifier-safe), 유저스페이스에서 조립
- /home 필터: 유저스페이스에서 처리 (커널 동적 인덱싱 verifier 거부 문제)
- Debounce: 10초 quiet period, 1시간 max wait
- Kafka: librdkafka, lz4 압축, message.timeout.ms=10s
- WAL: append-only, CRC32 체크섬, 시작 시 자동 replay

## 완료된 Phase
- Phase 1: PoC (eBPF probe + stdout 출력) ✅
- Phase 2: 전체 이벤트 감지 (vfs_utimes 추가) ✅

## 환경
- 개발 머신: Ubuntu 24, 커널 6.17.0-14-generic (sunday-desktop)
- sudo NOPASSWD 설정됨
- Kafka: Docker (apache/kafka:3.7.0, KRaft, localhost:9092)
- 토픽: file-tracker-events
- GitHub: regrow1123/file-tracker

## eBPF verifier 교훈
- 커널 내 동적 인덱싱(variable offset + variable size)으로 맵 접근 시 verifier 거부됨
- 해결: 고정 슬롯 배열(names[MAX_DEPTH][MAX_NAME_LEN])로 각 컴포넌트 개별 저장
- bpf_d_path()는 kprobe에서 사용 불가 (fentry/fexit만)
- per-CPU array map을 scratch buffer로 사용 (스택 512B 제한 회피)

## 2026-03-11 오전 세션 추가 진행

### rename 감지 구현 (commit 29737c8)
- vfs_rename kprobe 추가 (struct renamedata에서 old/new dentry 추출)
- EVENT_RENAME_FROM(2), EVENT_RENAME_TO(3) 이벤트 타입
- RHEL 9 VM에서도 빌드+실행 성공 (kprobe attach OK, bpf_events=12)
- 호스트 E2E 테스트 통과: mv → rename_from + rename_to 페어 감지

### rename 유저스페이스 페어링 (commit 3dafd1e)
- 소비자가 여럿일 때 rename_from/rename_to가 다른 파티션에 갈 수 있는 문제 지적됨
- 해결: 유저스페이스에서 ring buffer 콜백 순서로 from→to 페어링, 단일 Kafka 메시지로 전송
- JSON: `{"event":"rename", "old_path":"...", "new_path":"..."}`
- BPF에서 합치기 vs 유저스페이스 합치기 비교 분석 → 유저스페이스가 낫다 (ring buffer 크기, verifier 복잡도)

### Kafka 파티션/파티션 키 설명
- 현재 파티션 키 = hostname → 같은 노드의 이벤트가 같은 파티션 → 순서 보장
- 유저스페이스 페어링으로 파티션 키와 무관하게 안전

### 코드 품질 개선 (commit 004c3a0)
1. `system("mkdir -p ...")` → `mkdir(2)` 직접 호출 (command injection 방지)
2. JSON escape 강화: `\n`, `\t`, `\u0001`~`\u001f` 등 제어문자 전부 처리
3. global 변수 → `EventCtx` 구조체로 통합, `ring_buffer__new`에 ctx 전달
4. `cstdlib` 의존성 제거

### 제어문자 파일명 테스트 통과
- `tab\there.txt` → `\t` escaped
- `newline\nfile.txt` → `\n` escaped
- `ctrl\x01ugly.txt` → `\u0001` escaped
- 전부 유효한 JSON, 소비자가 원본 파일명 복원 가능
