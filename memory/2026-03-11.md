# 2026-03-11 파일변경추적 프로젝트 세션 기록

## 프로젝트 개요
- 대규모 리눅스 서버(수백대) 환경에서 /home 이하 파일 삭제/mtime 변경 실시간 추적
- eBPF 기반, 파일시스템 벤더 무관 (VFS 레벨)
- RHEL9 + RHEL10 동시 지원 (CO-RE 단일 바이너리)
- uid 추적 불필요, 파일 전체경로만
- Kafka로 이벤트 전달, WAL로 at-least-once 보장

## 기술 결정
- 언어: C++ (유저스페이스) + C (BPF probe)
- BPF: kprobe (vfs_unlink, vfs_write, do_truncate, vfs_utimes)
- 경로 해석: dentry 컴포넌트 배열 방식 (verifier-safe), 유저스페이스에서 조립
- /home 필터: 유저스페이스에서 처리 (커널 동적 인덱싱 verifier 거부 문제)
- Debounce: 10초 quiet period, 1시간 max wait
- Kafka: librdkafka, lz4 압축, message.timeout.ms=10s
- WAL: append-only, CRC32 체크섬, 시작 시 자동 replay

## 완료된 Phase
- Phase 1: PoC (eBPF probe + stdout 출력) ✅
- Phase 2: 전체 이벤트 감지 (vfs_utimes 추가) ✅
- Phase 3: Debounce + Kafka 연동 ✅
- Phase 4: WAL 구현 + 안정성 ✅
- 코드 리뷰: debouncer lock/unlock UB 수정, WAL replay 루프 방지 수정

## 환경
- 개발 머신: Ubuntu 24, 커널 6.17.0-14-generic (sunday-desktop)
- sudo NOPASSWD 설정됨
- Kafka: Docker (apache/kafka:3.7.0, KRaft, localhost:9092)
- 토픽: file-tracker-events
- GitHub: regrow1123/file-tracker

## eBPF verifier 교훈
- 커널 내 동적 인덱싱(variable offset + variable size)으로 맵 접근 시 verifier 거부됨
- 해결: 고정 슬롯 배열(names[MAX_DEPTH][MAX_NAME_LEN])로 각 컴포넌트 개별 저장
- bpf_d_path()는 kprobe에서 사용 불가 (fentry/fexit만)
- per-CPU array map을 scratch buffer로 사용 (스택 512B 제한 회피)
